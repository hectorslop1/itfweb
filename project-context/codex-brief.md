# Codex Brief

You are helping build a new website for Ives Torres Foundation.

## Project Summary

Ives Torres Foundation is a nonprofit supporting Hispanic families and people with intellectual and developmental disabilities across California.

The current site is built with Square/Weebly. We are rebuilding it from scratch as a modern, professional, warm, bilingual nonprofit website.

Do not copy the old Square website code.

Use the current site audit only as content and UX reference.

## Tech Stack

Use:
- Astro
- TypeScript
- Tailwind CSS
- React only where needed
- Sanity CMS
- Cloudflare Pages deployment target

## Design Requirements

The site should be:
- Modern
- Warm
- Professional
- Clean
- Accessible
- Mobile-first
- Bilingual-ready
- Emotionally engaging
- Fast
- SEO-friendly

The site should not:
- Feel like a store
- Use Square product/cart language
- Look like a generic template
- Be overly animated
- Be hard to update

## Main Pages

Create pages for:

- Home
- About
- Programs
- Events
- Resources
- Impact
- News
- Donate
- Contact

## Main Components

Create reusable components:

- Header
- Footer
- HeroSection
- SectionHeading
- Button
- ProgramCard
- EventCard
- NewsCard
- ResourceCard
- ImpactStats
- TestimonialCard
- FacebookHighlightCard
- DonateCTA
- ContactForm

## CMS Models

Create or plan Sanity schemas for:

- Event
- NewsPost
- Resource
- Program
- FacebookHighlight
- Testimonial
- Partner
- SiteSettings

## Homepage Structure

The homepage should include:

1. Hero with emotional message and CTAs
2. What we do
3. Featured programs
4. Upcoming events
5. Impact stats
6. Facebook highlights / community in action
7. Donation CTA
8. Resources preview
9. Contact CTA

## Donation Requirements

Donation page should support:

- PayPal as primary donation method
- Zelle as secondary donation method
- Clear donation instructions
- Suggested impact-based donation amounts
- Trust-building copy

## Facebook Section

Create a curated Facebook highlights section.

Each highlight should include:
- Image
- Title
- Description
- Link to Facebook post/video/event

Do not embed a full Facebook feed by default.

## Forms

Create contact form UI with fields:

- Name
- Email
- Phone optional
- Reason for contact
- Message

Possible reasons:
- I need support
- I want to volunteer
- I want to donate
- I represent an organization
- General question

## Accessibility

Use:
- Semantic HTML
- Accessible buttons and links
- Good color contrast
- Proper labels
- Keyboard-friendly navigation
- Respect reduced motion preferences

## Development Approach

Start with:

1. Project structure
2. Global styles
3. Layout
4. Header and footer
5. Homepage
6. Reusable cards
7. Main pages with placeholder content
8. Sanity schemas
9. CMS integration
10. Forms/donations
11. Final polish
